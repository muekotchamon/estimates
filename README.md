## GitHub Pages (Host บน GitHub)

- Push ขึ้น `main` แล้ว workflow จะ build และ deploy ให้อัตโนมัติ
- **สำคัญ:** ไปที่ repo → **Settings** → **Pages** → ใน **Build and deployment** ต้องเลือก **Source: GitHub Actions** (ห้ามใช้ "Deploy from a branch" เพราะจะได้หน้าเปล่า / 404 ที่ `/src/index.tsx`)
- หลัง deploy เสร็จ เปิดดูได้ที่: **https://muekotchamon.github.io/estimates/**

### ถ้าหน้าเว็บยัง blank / 404

1. **ตรวจว่าใช้ build จริง:** เปิด **https://muekotchamon.github.io/estimates/** → คลิกขวา → **View Page Source**  
   - ถ้าเห็น `src="/estimates/assets/index-...js"` = ใช้ผล build ถูกต้อง  
   - ถ้าเห็น `src="/src/index.tsx"` = ยังใช้ source จาก branch อยู่ → ไปที่ Settings → Pages เปลี่ยน Source เป็น **GitHub Actions**

2. **ให้ Pages ใช้ผลจาก workflow:** ไปที่ **Actions** → workflow "Deploy to GitHub Pages" → กด **Run workflow** (Run workflow) แล้วรอให้เสร็จ (สีเขียว)

3. **ล้าง cache:** เปิด URL ใน Incognito หรือกด Ctrl+Shift+R (hard refresh)
